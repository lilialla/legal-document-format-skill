# 安全政策

## 支持版本

| 版本 | 状态 |
|---|---|
| 1.x | 支持 |
| 0.x | 不再作为发布档维护 |

## 报告安全问题

请不要在公开 issue 中披露密钥、客户事实、真实案件材料或可识别个人/企业的敏感信息。优先使用 GitHub Security Advisory 私下报告；如果暂未启用 advisory，请先在公开 issue 中只说明“需要安全私下沟通”，不要粘贴敏感内容，等待维护者提供私有渠道。

报告时请尽量包含：

- 受影响文件或命令；
- 可复现步骤；
- 风险影响；
- 是否涉及真实材料、密钥或第三方平台导出。

## 敏感信息处理原则

如果发现敏感信息进入仓库：

1. 立即停止继续传播。
2. 轮换受影响密钥或令牌。
3. 移除 tracked 文件中的敏感内容。
4. 增加测试、文档或门禁，避免同类问题复发。

## 项目边界

本项目只处理本地文件和 synthetic 示例。它不提供托管服务，不要求上传法律材料，也不应保存真实案件、客户信息或私有模板。
